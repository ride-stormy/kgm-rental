# CPS: kgm-rental-platform

> Context → Problem → Solution 프레임워크
> 이해관계자 간 싱크를 맞추고, 프로젝트 방향성을 명확히 하기 위한 문서.

## Context (맥락)

### 비즈니스 배경

- **프로젝트 목표**: KGM 장기렌트 **400대 한정** 단기 집중 판매 (속도 중심, 매출 확보 최우선).
- **운영 주체**: 라이드㈜(KGM 공식 리테일러, KGM Experience Center 강남/일산/부산 운영) × **메리츠캐피탈** 독점 제휴.
- **핵심 차종**: 토레스(월 39만원 장기렌트), 액티언 하이브리드(월 18만원/36개월).
- **정책 제약**: "최저가", "전국 최저" 등 표현 금지. 브랜드 정책 위반 요소 배제.
- **핵심 메시지**: 효율보다 속도, 완성도보다 전환율. 잘하는 프로젝트가 아니라 살아남는 프로젝트.

### 현재 시스템 상태

- **유입 구조(기존, 유지)**: 유튜브 영상 시청 → 전화 문의 → 상담 → 계약.
- **견적 도구**: 메리츠캐피탈 배포용 xlsm(`메리츠캐피탈 렌터카 견적시트_2604.v3`) — 14개 시트(견적조건/차량정보/탁송/정비/렌트 입력·출력 등) 수식 기반으로 영업사원이 **수기 계산**.
- **세일즈 어드민**: 기존 시스템 존재. 이번 프로젝트에서 **새로 만들지 않음**. 상담 CTA 클릭 시 해당 어드민으로 **딥링크 전달**하는 접점만 필요.
- **재고 관리**: 400대 잔여 수량을 조회하는 **재고 API**는 **백엔드 개발자(외부 팀)가 전달**. 이번 피처에서 구현하지 않음.

### 이해관계자

| 이해관계자 | 역할 | 핵심 관심사 |
|-----------|------|-----------|
| 콘텐츠 팀 | 유튜브 영상 제작(단일, 장기렌트 중심), 더보기 CTA 연동 | 영상 클릭 → 랜딩 진입 전환율 |
| 세일즈 팀 | 상담 스크립트 작성·확정, 상담 및 계약 처리 (기존 어드민 사용) | 인입 고객의 정확한 상품·조건 컨텍스트 수신 |
| 프로덕트 팀 (우리) | B2C 장기렌트 페이지 + 조건별 가격 계산기 + 상담 신청 구축 | 유입 → 조건 탐색 → 상담 신청까지 마찰 없는 동선 |
| 백엔드 개발자(외부) | 재고 API 제공 | 400대 잔여 수량 실시간 동기화 |
| 메리츠캐피탈 | 렌터카 특별 한정 상품·견적 로직 공급 | xlsm 기반 견적 계산식의 정확한 이식 |

| 고객 | 영상 시청 후 장기렌트 상품을 탐색하고 견적을 확인 | 조건 변경 시 즉시 반영되는 월 렌트료, 빠른 상담 연결 |

## Problems (문제)

### P1: 영상 시청 후 구매 타이밍 이탈

- **페르소나**: 고객(유튜브 시청 후 관심 발생한 잠재 구매자)
- **현상**: 영상 시청 후 전화 문의를 기다리는 구조라 관심이 식기 전에 대응하지 못함. 영상 더보기 → 외부 단순 문의 폼 구조에서 조건 탐색 없이 이탈.
- **영향**: 400대 한정 단기 집중 판매 목표의 전환율이 저하되어 완판 기간이 늘어남.
- **근본 원인**: "영상 보고 전화 기다리는 구조"는 구매 타이밍을 실시간으로 포착하지 못함.

### P2: 상품·조건 탐색 경로 부재

- **페르소나**: 고객
- **현상**: 고객이 토레스/액티언 HEV 등 상품을 한눈에 비교하고 본인 상황(계약기간, 주행거리, 정비 패키지 등)에 맞춘 월 렌트료를 직접 확인할 경로가 없음.
- **영향**: 모든 질문이 상담 전화로 몰려 상담 시간이 길어지고 이탈률이 증가. 상담 없이 결정을 내릴 수 있는 고객까지 상담 큐에 누적됨.
- **근본 원인**: 상품 카탈로그와 조건별 가격 계산 기능이 공개 웹에 존재하지 않음.

### P3: 수기 견적 지연 — xlsm 의존

- **페르소나**: 영업사원 / 고객(상담 중)
- **현상**: 14개 시트로 구성된 메리츠 xlsm을 영업사원이 열어 입력·복사·검토해야 월 렌트료가 나옴. 계약기간/주행거리/정비 조합이 바뀔 때마다 반복.
- **영향**: 상담 통화가 길어짐 → "상담은 짧고 빠르게"라는 운영 원칙과 충돌. 고객이 조건 비교를 요청하면 대응 속도가 급락.
- **근본 원인**: 견적 계산 로직이 수식 스프레드시트에 갇혀 있어 웹/앱으로 호출 가능한 형태로 노출되지 않음.

### P4: 상담 신청 컨텍스트 손실

- **페르소나**: 세일즈 팀(기존 어드민 사용자)
- **현상**: 고객이 상담을 신청해도 "어떤 차종, 어떤 조건(계약기간·주행거리·정비·선택형/인수형), 어떤 예상 월 렌트료"를 보고 있었는지 세일즈 어드민에 전달되지 않음.
- **영향**: 상담 첫 30초에 상황을 다시 파악해야 함 → 통일된 스크립트("상황 파악 → 바로 추천 → 가격 제시 → 계약 유도") 실행이 어려움.
- **근본 원인**: B2C 페이지가 상담 신청을 외부 어드민으로 넘길 때 상품·조건·견적 스냅샷을 함께 전달하는 인터페이스가 정의되지 않음.

## Solutions (해결책)

### S1: 장기렌트 상품 진열 페이지 ← P1, P2

- **접근 방식**:
  - 유튜브 영상 "더보기" CTA가 진입하는 **B2C 웹페이지**를 구축.
  - 토레스/액티언 HEV 등 **특별 한정 상품을 카드 형태로 진열**. 차종·차량 이미지·핵심 조건(예: "월 18만원 / 36개월")·하단 CTA.
  - 상품 상세에서 차량 사양·장기렌트 장점(취등록세·세금·보험·관리 포함) 요약. 금지 표현("최저가", "전국 최저") 필터링.
  - 외부 재고 API로 **400대 중 잔여 수량을 배지/배너로 노출**해 긴급성 유지(API는 외부 팀 제공, 우리는 호출만).
- **기술 스택**: Next.js 15(App Router, SSR/ISR) · React 19 · Tailwind CSS · shadcn/ui · TanStack Query v5(재고 폴링) · ky(HTTP).
- **예상 결과**: 유튜브 → 페이지 진입 후 상품을 본인 눈으로 확인하고 다음 단계(견적 계산)로 자연스럽게 이동. 상담 전화 의존 감소.
- **제약 조건**: 브랜드 정책 위반 문구 금지. 단일 영상 → 단일 랜딩 경로라는 운영 원칙에 맞춰 정보 밀도를 낮춤(스크롤 깊이 최소화).

### S2: 조건별 렌트 가격 계산기 ← P2, P3

- **접근 방식**:
  - 메리츠캐피탈 xlsm의 견적 수식(견적조건·차량정보·탁송·정비·렌트 입력·렌트 출력 시트)을 **도메인 로직으로 재구현**.
  - 사용자 입력: ① 차종/모델, ② 계약기간(36/48/60), ③ 약정 주행거리(20k/10k/15k/25k/30k/40k/무제한), ④ 정비 상품(Basic/Standard/Select/Platinum), ⑤ 만기 방식(선택형/인수형), ⑥ 월동장비(체인/타이어), ⑦ 지역(탁송료).
  - 출력: **월 렌트료, 보증금, 선납, 초기 부담금액, 만기 시 예상 인수가**. 입력이 바뀌면 실시간 재계산.
  - xlsm 기반 **기준 값(출고년월·보조금·개별소비세 감면·전기차 보조금 등)은 서버 측 레퍼런스 데이터로 관리**해 변경 시 배포 없이 교체 가능하게.
- **기술 스택**: NestJS 10(계산 서비스를 pure 도메인 서비스로) · TypeORM(기준값 저장) · PostgreSQL · Zod(입력 검증) · TanStack Query(프론트 캐싱) · React Hook Form + Zod(입력 폼).
- **예상 결과**: 고객이 **상담 없이도 본인 조건의 월 렌트료를 즉시 확인**. 영업사원은 상담에서 xlsm을 열지 않고 고객이 이미 본 조건을 그대로 확인.
- **제약 조건**: 메리츠 xlsm 수식과 계산 결과가 **원본과 일치**해야 함(샘플 견적 스냅샷으로 회귀 테스트). 조건 조합에 따른 엣지케이스(EV 보조금, 개소세 감면, 한정 프로모션)는 기준값 테이블로 분리.

### S3: 상담 신청 연동 (기존 세일즈 어드민 딥링크) ← P1, P4

- **접근 방식**:
  - "상담 신청" CTA 클릭 시 고객이 **현재 보고 있는 상품·조건·계산 결과 스냅샷**을 백엔드에 저장하고, 저장된 식별자를 포함한 **기존 세일즈 어드민의 딥링크**로 이동시키거나 세일즈 어드민에 알림 전송.
  - 수집 항목: 성함/연락처(카톡 ID 선택), 선호 지점(강남/일산), 선호 연락 채널(전화/카톡), 상품·조건·견적 스냅샷.
  - 개인정보 동의·이용약관 체크(필수). PII 최소 수집.
- **기술 스택**: NestJS 10(Consultation 도메인) · PostgreSQL(consultation_requests) · React Hook Form + Zod(폼) · ky(외부 세일즈 어드민 웹훅/딥링크).
- **예상 결과**: 상담 첫 30초에 세일즈가 이미 고객 상황을 파악한 상태에서 "상황 파악 → 바로 추천 → 가격 제시" 스크립트를 곧바로 실행. 상담 시간 단축, 전환율 상승.
- **제약 조건**: 세일즈 어드민의 외부 인터페이스 스펙(딥링크 형식/웹훅 payload)은 세일즈 팀·외부 백엔드 개발자와 Plan 단계에서 합의 필요. 개인정보 수집·이용·제3자 제공 동의 문구 법무 확인 필요.

## 이번 피처 범위 정리

**포함(In Scope — `kgm-rental-platform`)**
- **S1**: B2C 장기렌트 상품 페이지(진열·상세)
- **S2**: 조건별 가격 계산기(메리츠 xlsm 로직 이식) — **최우선**: 계산 결과가 xlsm 원본과 **반드시 일치**해야 함. 샘플 견적 스냅샷 기반 회귀 테스트 필수.

이번 피처의 목적은 **"유튜브에서 인입된 고객이 상품을 확인하고 본인 조건의 월 렌트료를 즉시·정확하게 확인하는"** 경험을 빠르게 만드는 것. 기획 변경 가능성이 있으므로 **빠른 가시화**가 중요.

**다음 피처(Out of Scope — 별도 Plan)**
- **S3 → `kgm-rental-consultation`**: 상담 신청 폼 + 스냅샷 저장 + 기존 세일즈 어드민 딥링크/웹훅. S1/S2가 시장 검증된 후 착수.

**본 프로젝트 범위 외(Not Built Here)**
- 400대 재고 관리 시스템(외부 백엔드 팀이 API로 제공, 우리는 호출만)
- 세일즈 어드민(이미 존재, 수정하지 않음)
- 400대 판매 현황 관제 대시보드(별도 Epic 후보)
- 유튜브 영상 제작(콘텐츠 팀)

## 품질 기준 (Plan으로 넘길 때 강조 필요)

1. **S2 견적 계산 정확도 100%**: xlsm의 대표 조건 조합(차종×계약기간×주행거리×정비×만기방식) 샘플 견적을 수집해 회귀 테스트. 결과가 원본과 **1원이라도 다르면 실패**.
2. **빠른 가시화**: 기획 변경 가능성 있음 → Plan에서 **얇은 수직 슬라이스**(1차종 + 소수 조건 조합)로 끝까지 동작하는 프로토타입을 먼저 가동하고 확장.
3. **금지 표현 필터**: "최저가", "전국 최저" 등 브랜드 정책 위반 문구가 카피·카드·메타에 포함되지 않도록 렌더 시 검증.
