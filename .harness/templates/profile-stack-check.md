# Profile Stack Check — 프로필 기술 스택 불일치 검증

> 프로필이 설정된 프로젝트에서 기술 스택 관련 결정이 발생할 때 이 절차를 따른다.

## 실행 조건

- `.harness/config.json`에 `project.profile`이 설정되어 있을 때
- 프로필이 없으면 이 절차를 건너뛴다

## 절차

### 1. 프로필 로드

`.harness/config.json`의 `project.profile` 값으로 프로필을 로드한다:
- `.harness/profiles/{name}/profile.json` 직접 Read (로컬 우선)
- 없으면 `.harness/lib/profile-loader.js`의 `loadProfile(name, projectRoot)` 실행

### 2. 스택 비교

현재 단계에서 사용/언급된 기술 목록과 프로필의 `stack` 필드를 비교한다.

| 단계 | 비교 대상 |
|------|----------|
| Discovery | 회의록에서 추출한 기술 |
| CPS | Solution에 명시된 기술 |
| Plan | Feature 분해에 사용된 기술 |
| Design | 설계안에 포함된 프레임워크/라이브러리 |

### 3. 불일치 감지 시 AskUserQuestion

불일치가 발견되면 **반드시** 아래 형태로 사용자에게 확인한다:

```
⚠️ 프로필 스택 불일치 감지

| 항목 | 프로필 (profile.json) | 현재 문서에서 감지 |
|------|---------------------|-----------------|
| frontend | React + TypeScript | Expo (React Native) |
| ... | ... | ... |

어떻게 처리할까요?
1. 프로필 스택 유지 → 현재 문서를 프로필 기준으로 수정
2. 새 기술 채택 → 프로필을 업데이트하고 진행
3. 병행 사용 → 프로필에 추가하고 진행
```

실제 값으로 테이블을 채워서 표시한다.

### 4. 처리 결과 반영

| 사용자 선택 | 처리 |
|-----------|------|
| 1. 프로필 스택 유지 | 현재 문서/추출 결과를 프로필 `stack` 기준으로 재작성 |
| 2. 새 기술 채택 | 진행 후 세션 마지막에 프로필 업데이트 안내: "프로필에 {기술}을 추가하려면 `/harness:profile` 명령을 사용하세요." |
| 3. 병행 사용 | 프로필의 해당 `stack` 필드에 추가하고 진행 |

### 5. 금지 사항

- AI가 임의로 프로필을 무시하고 다른 기술을 채택하는 것을 금지
- AI가 확인 없이 프로필 우선을 적용하는 것도 금지
- 사용자 확인 없이 기술 스택을 결정하지 않는다
