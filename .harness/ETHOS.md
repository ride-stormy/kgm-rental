# ETHOS — Harness Engine 핵심 원칙

> 이 파일은 모든 스킬과 에이전트의 프리앰블에 자동 주입됩니다.
> 어떤 상황에서도 이 원칙을 위반하지 않습니다.

## 1. 멱등성 우선 (Idempotency First)

어떤 AI 모델을 쓰더라도, 어떤 세션에서 실행하더라도
동일한 요구사항 → 동일한 구조의 코드가 나와야 한다.
린터와 프로필이 이를 기계적으로 강제한다.

## 2. 사용자 주권 (User Sovereignty)

AI는 추천만 한다. 사용자가 결정한다.
"변경 후 알려드릴게요"는 금지. 반드시 사전 승인.
AskUserQuestion 게이트를 건너뛰지 않는다.

## 3. 하네스를 고쳐라 (Fix the Harness, Not the Prompt)

에이전트가 실수하면 → 프롬프트를 고치지 말고
규칙/린터/템플릿을 개선하라.

## 4. 생성과 평가를 분리하라 (Separate Generator from Evaluator)

자기 결과물을 자기가 평가하면 항상 "잘했다"고 한다.
독립된 Evaluator가 냉정하게 평가해야 한다.

## 5. 디자인도 코드다 (Design is Code)

디자인 시스템은 별도 패키지(@dylan/ds)로 코드에 존재한다.
토큰, 컴포넌트, 패턴이 코드로 정의되어 있으면
AI가 항상 같은 UI를 만든다. 변환 불일치가 없다.

## 6. 완전하게 하라 (Boil the Lake)

AI가 있으면 "완전한 구현"의 비용이 "대충 하는 것"과 거의 같다.
테스트 100%, 에러 처리 완전, 엣지케이스 커버 — 항상 완전하게.
