## 2. xlsm 캐시 상태 자동 해독 (case_id=1 완성)

추출 스크립트가 xlsm 캐시에서 읽은 마지막 계산 상태를 분석하여 golden CSV의 **case_id=1** 행을 자동 완성했다.

### 입력 조건 (xlsm에 저장된 현재 상태)

| 항목 | 값 | 출처 셀 |
|------|-----|---------|
| 모델 | 액티언 1.5T HEV | `렌트_입력시트!EO8` |
| 차량가 (기본, X8) | 40,690,000 | `렌트_입력시트!X8` |
| 차량가 (할인후) | 37,790,000 | `1 견적조건!H4` |
| 일반할인 | 2,900,000 | `1 견적조건!H9` |
| 특판할인 | 772,000 | `1 견적조건!H8` |
| 계약기간 (조건1) | 36개월 | `1 견적조건!M4` |
| 약정주행거리 (조건1) | 년 20,000km | `1 견적조건!M10` |
| 선납률 | 30% | `렌트_입력시트!BR21` |
| 보증금률 | 0% | `렌트_입력시트!BR8` |
| 잔가율 | 72% | `렌트_입력시트!BR9` |
| 적용금리 (BR32) | 5.2957% | `렌트_입력시트!BR32` |
| 기준IRR | 5.6% | `렌트_입력시트!BR34` |

### 출력 (계산 결과)

| 필드 | 값 | 출처 셀 |
|------|-----|---------|
| 공급가 | 480,200 | `렌트_입력시트!BR26` |
| 부가세 | 48,020 | `렌트_입력시트!BR27` |
| 월렌트료 ① (표준) | 528,220 | `렌트_입력시트!BR23` |
| 선납렌트료 ② (차감) | -339,080 | `렌트_입력시트!BR28` |
| **최종 월 렌탈료** | **189,140** | `렌트_출력시트!Y30` |
| 잔가 (= 인수가) | 29,297,000 | `렌트_입력시트!CC9` |
| 선납금액 | 12,207,000 | `렌트_입력시트!CC21` |
| 보증금액 | 0 | `렌트_입력시트!CC8` |
| 초기부담금 | 12,207,000 | `렌트_출력시트!Y33` |

### 수학 검증 (상호 일관성)

- `BR23 = BR26 + BR27` → 528,220 = 480,200 + 48,020 ✓
- `Y30 = BR23 + BR28` → 189,140 = 528,220 + (-339,080) ✓
- `BR28 = -ROUNDDOWN(CC21 / 36, -1)` → -339,080 ✓
- `CC21 = ROUNDUP(X8 × 0.3, -3)` → 12,207,000 ✓
- `CC9 = ROUNDUP(X8 × 0.72, -3)` → 29,297,000 ✓
- `BR26 = ROUNDUP(EG7+EG8+EG9+EG10+EG11+EG12+EG13, -2)` → ROUNDUP(480,170, -2) = 480,200 ✓

## 3. Design AC 정정 필요

Design `docs/02-design/features/quote-engine.design.md` § 5.5는 다음 문구를 포함한다:

> 베스트: 액티언HEV 하이브리드 S8 36M/20k/**0/0** → 최종 월 189,140원 ≈ 18만원 재현

수학적으로 `0/0`(선납·보증 없음)이면 `CC21 = 0`, `BR28 = 0`, 최종 = `BR23 = 528,220`이 나와야 한다. 즉 **189,140원은 `0/0` 상태가 아니라 `30/0` 상태의 값**이다.

**정정 제안(Stage 2 초반 작업):** Design AC를 `액티언HEV 36M/20k/30/0 = 189,140원`으로 수정.

## 4. golden CSV 현황

`apps/backend/libs/src/modules/rental-quote/domain/domain-services/test/__fixtures__/golden-quotes.csv`에 25건의 case 골격을 만들었다.

| 상태 | case_id 범위 | 수량 |
|------|--------------|------|
| expected_* 완성 | 1 | 1건 |
| expected_* 공란 (사용자가 xlsm에서 재계산 후 채워야 함) | 2~25 | 24건 |

### case_id=2~25 채우는 작업 방법 (사용자 수작업)

1. Excel(또는 LibreOffice Calc)에서 `pre-docs/메리츠캐피탈 렌터카 견적시트_2604.v3_배포용.xlsm` 열기
2. case별로 다음 4개 셀 입력:
   - `1 견적조건!M4` = 계약기간 (24/36/48/60)
   - `1 견적조건!M10` = 약정주행거리 (년 10,000km 선택 등)
   - `렌트_입력시트!BR21` = 선납률 (0/0.1/0.2/0.3)
   - `렌트_입력시트!BR8` = 보증금률 (0/0.1/0.2/0.3)
   - (차량 변경 시: `렌트_입력시트!EO8` 모델 + 가격 재설정)
3. 저장 시 재계산된 다음 9개 값을 CSV에 기입:
   - `렌트_입력시트!BR23/BR26/BR27/BR28/CC8/CC9/CC21`
   - `렌트_출력시트!Y30/Y33`
4. case_id=17(30%/30%)은 도메인 예외 케이스 → expected_* 공란 유지

> **참고**: 스크립트 재실행으로 1건씩 자동 추출도 가능하다. `extract-xlsm.mjs`에 `--probe-only` 모드 추가하면 xlsm 저장 상태의 probe 값만 빠르게 읽어올 수 있다. (Stage 2에서 도구 개선 고려)

## 6. 불확실·누락 항목

1. **H21 (취득원가)**: `readCell(inp, 'H', 21)` 결과 null. 수식 `sheet2!M49`의 `PMT(rate, nper, -H21, ...)`에서 H21이 어느 시트인지 재확인 필요. 후보:
   - `1 견적조건!H14 = 35,119,063` (공급가격)
   - `렌트_입력시트!AL34 = 35,218,372` (취득원가 라벨)

2. **BR32 이율 결정 로직**: 현재 BR32 = 5.2957%. 차종×계약기간 매트릭스 lookup 결과인지, 단일 기준값(BR34=5.6%)에서 파생된 것인지 수식 확인 필요.

3. **개별소비세 감면**: `EG24 = 1,211,903.42` 캐시됨. HEV 개소세 감면 분기의 수식 확인 필요.

4. **EV 특판 150만원 분기**: Design에서 언급. 실제 xlsm 어느 셀에 분기 수식이 있는지 확인 필요.
